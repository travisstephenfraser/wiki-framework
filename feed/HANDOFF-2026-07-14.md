## Key decisions (do not relitigate)

- **Ledger over recompute.** Confidence is a human-approved judgment sealed with a content fingerprint, never recomputed from source strings. Two independent measurers (fork red-team, upstream) reached this; it is settled.
- **Atomic fields, staged approvals.** trust-check fail-closes if any content page lacks the schema; only ledger approval is incremental. Empirically verified, not a preference.
- **Never `pip install obsidian-wiki`; never run setup.sh GitHub-sync.** CLI runs from the checkout: `cd <repo> && python3 -m obsidian_wiki <cmd>`.
- **`stale` means re-review, not error.** Material edit to a ledgered page reopens review; re-record after human sign-off. Timestamp-only churn never does.

## The checkpoint (the actual task)
